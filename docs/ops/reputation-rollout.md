# Reputation Rollout

## Цель

Построить репутацию как историю качества диалога, а не как счётчик побед, лайков или абстрактный рейтинг силы.

## Статус блока

- Стадия: `implemented_v1`
- Ближайший релиз: `v1 public safe badges`
- Возвращаться к блоку через этот файл и обновлять в нём выполненные шаги до синхронизации `status/roadmap`

## Зачем это нужно

- это ключевой дифференциатор Konsensus;
- репутация должна поощрять конструктивность, а не агрессию;
- она должна объяснять, как человек ведёт диалог, а не “кто лучше”.

## Решение v1

Первый слой:
- приватный AI-профиль остаётся полным;
- публичный слой получает только безопасные стилевые бейджи;
- никаких токсичных публичных ярлыков.

Примеры бейджей:
- `Дипломат`
- `Рациональный`
- `Слышащий`
- `Настойчивый`
- `Аккуратен с фактами`

## Выполнено в v1

- Добавлен rule-based helper публичной репутации без отдельного score-движка
- Без новой БД-схемы собраны безопасные бейджи на базе существующих сигналов:
  - `compromise_tendency`
  - `consensus_rate`
  - `empathy_score`
  - `argumentation_style`
  - реакция на AI hints
  - evidence-based аргументы
  - RPG persistence / activity
- Бейджи выведены:
  - в личном профиле внутри вкладки `AI-профиль`
  - в публичной hover-карточке автора на арене
- Модель остаётся безопасной:
  - нет leaderboard
  - нет общего числа “силы”
  - нет публичных негативных ярлыков

## Источники сигналов

На первом этапе использовать:
- `dispute_analysis`
- `round_insights`
- долю завершённых консенсусом споров
- реакцию на AI hints
- частоту evidence-based аргументов

## Встраивание в текущий код

### v1
- ввести mapping `signals -> public badges`
- не строить сложный рейтинг-балл
- показывать на публичном профиле:
  - активные бейджи;
  - краткое объяснение за что они получены

### Data model
- можно использовать существующие achievement / profile tables на первом этапе
- если потребуется явная история, добавить отдельную таблицу `user_reputation_badges`

### Данные и сервер

- v1 можно начать без отдельного reputation score
- считать набор безопасных badge-сигналов на базе уже существующих dispute данных
- если понадобится persistence, добавить `user_reputation_badges` с историей назначения и источником сигнала

### UI-поверхности

- блок публичных бейджей в профиле
- краткое объяснение каждого бейджа
- без негативных маркеров и без leaderboard

### Критерий готовности v1

- пользователь получает ограниченный набор позитивных/нейтральных бейджей
- бейджи объяснимы через dispute signals
- модель не создаёт токсичных ярлыков
- это можно расширить до reputation history и appeals без переписывания слоя

## Что пока не делать

- leaderboard “кто лучший спорщик”
- единый score “силы”
- публичные негативные ярлыки
- сложные проценты, которые пользователь не понимает
