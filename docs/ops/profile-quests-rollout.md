# Profile Quests Rollout

## Цель

Добавить игровое профилирование пользователя без тяжёлого игрового движка и без длинных анкет.

Первый шаг — короткие текстовые сценарии выбора, которые обновляют внутренний AI-профиль пользователя и постепенно улучшают персонализацию mediation-flow.

## Зачем это нужно

- стандартные анкеты дают социально желаемые ответы;
- поведенческие выборы дают более полезный сигнал;
- AI-подсказки становятся персональными, а не усреднёнными;
- это усиливает и dispute core, и будущий образовательный слой.

## Решение v1

Ввести раздел `Квесты профиля`:
- 3 коротких сценария;
- каждый сценарий занимает 2-4 минуты;
- текст + 2-4 ветвящихся выбора;
- без сложной графики, без мини-движка.

## Какие сигналы снимать

На первом этапе достаточно обновлять:
- `compromise_tendency`
- `impulsivity`
- `empathy_score`
- `argumentation_style`

Допустимо хранить промежуточные выборы, но для пользователя показывать только обновлённый приватный профиль.

## Встраивание в текущий код

### v1
- использовать уже существующую таблицу `user_ai_profiles`
- добавить таблицу или lightweight-лог для прохождения квестов, если нужен audit trail
- новые server actions:
  - `startProfileQuest`
  - `submitProfileQuestChoice`
  - `completeProfileQuest`

### UI
- отдельная страница в профиле или dashboard
- карточки сценариев: `Реакция на давление`, `Несправедливость`, `Дефицит информации`
- после завершения показывать:
  - что изменилось в AI-профиле;
  - как это влияет на рекомендации ИИ.

## Что пока не делать

- полноценный narrative engine
- десятки квестов
- графические сцены
- публичное сравнение профилей между пользователями
