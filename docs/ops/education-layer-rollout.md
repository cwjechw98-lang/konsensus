# Education Layer Rollout

## Цель

Превратить спор в точку роста пользователя: после конфликта человек должен получать не только результат, но и понятный следующий шаг для развития навыков диалога.

## Статус блока

- Стадия: `planned`
- Ближайший релиз: `v1 markdown materials + rule-based recommendations`
- Возвращаться к блоку через этот файл и обновлять в нём выполненные шаги до синхронизации `status/roadmap`

## Зачем это нужно

- повышает retention не только через конфликты;
- делает AI-профиль practically useful;
- создаёт основу для premium и внутренней образовательной экономики.

## Решение v1

Минимальный образовательный слой:
- 5-10 коротких материалов в Markdown;
- темы: риторика, деэскалация, переговоры, активное слушание;
- персональные рекомендации после спора или на dashboard.

## Как это должно работать

После спора система может показать:
- что у пользователя получилось;
- что было слабым местом;
- какой короткий материал поможет улучшиться.

Пользователь видит:
- `Следующий полезный шаг`
- 1-2 рекомендованных материала
- отметку о прохождении

## Встраивание в текущий код

### v1
- хранить материалы как Markdown в `docs/` или `content/`
- добавить lightweight metadata:
  - тема
  - skill tags
  - кому полезно
- показывать рекомендации в profile/dashboard

### Recommendation logic
- на первом этапе rule-based:
  - высокая импульсивность -> деэскалация
  - низкая evidence usage -> аргументация и факты
  - низкий consensus rate -> переговоры и common ground

## Пакет реализации v1

### Основные файлы

- новый каталог контента, например `content/education/`
- dashboard/profile страницы, где уже есть AI-профиль и post-dispute surface
- [src/lib/actions.ts](/C:/project21/konsensus/src/lib/actions.ts) или профильный loader
- [src/types/database.ts](/C:/project21/konsensus/src/types/database.ts), если добавлять историю прохождения

### Данные и сервер

- материалы хранить как Markdown с frontmatter metadata
- recommendation logic сделать rule-based поверх текущих AI-profile и dispute signals
- при необходимости добавить lightweight-таблицу `user_learning_progress`

### UI-поверхности

- блок `Следующий полезный шаг` на dashboard или в профиле
- список из 1-2 рекомендованных материалов
- простой экран чтения материала
- отметка о прохождении

### Критерий готовности v1

- есть 5-10 коротких материалов
- рекомендации выдаются без AI-поиска и без сложной LMS
- пользователь видит, почему материал предложен именно ему
- прохождение можно позже связать с achievements и profile quests

## Что пока не делать

- полноценную LMS
- видео и сертификаты
- сложную валютную экономику
- редактор курсов для авторов
