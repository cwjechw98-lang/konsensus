# Activity Feed Rollout

## Цель

Превратить раздел `События` из простого списка публичных споров в реальный activity feed, который:
- показывает жизнь продукта без длинных пояснений;
- даёт быстрый срез по релизам, live-арене и публичным спорам;
- усиливает ощущение, что проект живой и в нём есть движение.

## Статус блока

- Стадия: `implemented_v1`
- Текущий релиз: `Activity Feed v1`
- Возвращаться к блоку через этот файл и обновлять здесь выполненные шаги, затем синхронизировать `docs/status.md` и `docs/roadmap.md`

## Выполнено в v1

- `/feed` перестроен из списка публичных споров в sectioned activity feed
- Добавлен верхний summary-блок по основным потокам:
  - релизы продукта
  - live battle на арене
  - открытые вызовы
  - публичные споры
- Добавлен отдельный блок релизов на базе `release_announcements`
- Добавлен основной поток активности:
  - активные live battle
  - открытые вызовы арены
  - публичные споры
- `События` получили более точный contextual tour под новый смысл раздела
- Релизы в feed читаются серверно через admin client без ослабления публичных RLS-политик

## Продуктовые решения v1

### 1. Роль раздела

`События` — это не рабочий список пользователя и не каталог вступления.

Его роль:
- показать, что сейчас происходит в проекте;
- дать быстрый вход в наблюдение;
- визуально доказать, что продукт живой.

### 2. Источники данных v1

В первый релиз feed собирается без новой БД-схемы из уже существующих источников:
- `release_announcements`
- `challenges` со статусами `active` и `open`
- публичные `disputes`

### 3. Структура экрана

Утверждённая структура:
1. context-card
2. summary grid
3. блок релизов
4. основной activity stream

### 4. Что не делаем в v1

- не строим ranking system
- не добавляем персонализацию ленты
- не добавляем комментарии/реакции прямо в feed-карточки
- не делаем отдельные release detail pages

## Критерий готовности v1

- раздел `События` больше не выглядит пустой витриной одного типа контента;
- пользователь сразу видит:
  - что нового в продукте;
  - где идёт live-активность;
  - какие публичные споры сейчас заметны;
- feed лучше объясняет идею проекта через движение, а не через длинные тексты.
