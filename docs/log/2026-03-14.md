# Лог разработки — 2026-03-14

## Сессия 2: Критические фиксы + AI coaching + Медиация

### Что сделано

#### Критические баги
- **Rate limiting** в `createDispute`: максимум 5 активных споров на пользователя
- **Валидация входных данных**: название минимум 3 символа, описание минимум 5 символов
- **AI error states**: `triggerMediation` больше не крашится если Groq недоступен — сохраняет fallback-сообщение и завершает медиацию

#### Принятие решений в медиации (`CRITICAL #1`)
- Добавлено server action `acceptSolution` — участники могут принять одно из предложенных ИИ решений
- Создан `MediationClient.tsx` — клиентский компонент с real-time обновлениями через Supabase Realtime
- Когда оба участника принимают одно решение → статус `"accepted"`, отображается баннер "Консенсус достигнут! 🎉"
- UI показывает кто уже принял какое решение в реальном времени

#### AI waiting insight (подсказки ИИ во время ожидания)
- **Новая таблица** `waiting_insights` (миграция `docs/migration-waiting-insights.sql`)
- Функция `generateWaitingInsight` в `ai.ts` — генерирует персональную подсказку для участника сразу после его отправки аргумента
- Подсказка объясняет ПОЧЕМУ оппонент думает иначе — coaching без суждений, 2-3 предложения
- Отображается в `RealtimeDisputeClient` в "waiting" состоянии под "Ждём ответа..."
- Исчезает автоматически когда оппонент ответил
- Real-time обновление: новая подсказка появляется через Supabase Realtime без перезагрузки страницы

### Миграции для запуска
```sql
-- docs/migration-waiting-insights.sql
-- Запустить в Supabase SQL Editor
```

### Решения
- `acceptSolution` использует `createAdminClient()` для обхода RLS (т.к. RLS для resolutions может не быть настроен)
- `generateWaitingInsight` вызывается синхронно в `submitArgument` — добавляет ~1-2 сек к отправке аргумента, приемлемо для MVP
- `existingArgs` расширен до `author_id, round, position, reasoning, evidence` для контекста ИИ-подсказки
