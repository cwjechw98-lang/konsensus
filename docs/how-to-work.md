# Как работать с проектом Konsensus

Инструкция для пользователя по работе с проектом через Claude Code.

## Быстрый старт: возобновление работы

### 1. Открыть проект
```bash
cd C:\project21\konsensus
claude
```

Claude Code автоматически прочитает `.claude/CLAUDE.md` и будет в контексте проекта.

### 2. Возобновить работу
Отправь одну из команд:

**Продолжить с того места, где остановились:**
```
Продолжи работу над проектом. Проверь текущий статус в CLAUDE.md и docs/roadmap.md, посмотри последний лог в docs/log/ и предложи следующие шаги.
```

**Начать конкретную задачу:**
```
Начни работу над [название фазы/задачи из roadmap.md]. Сначала проверь текущий статус проекта.
```

**Проверить статус:**
```
Покажи текущий статус проекта: что сделано, что в процессе, что дальше.
```

## Полезные команды для ИИ

### Работа с кодом
| Команда | Что делает |
|---------|-----------|
| `Создай компонент [название]` | Создаёт React-компонент по конвенциям проекта |
| `Добавь страницу [путь]` | Создаёт новую страницу в App Router |
| `Настрой Supabase` | Подключает Supabase клиент, создаёт миграции |
| `Добавь аутентификацию` | Настраивает Supabase Auth |

### Управление проектом
| Команда | Что делает |
|---------|-----------|
| `/log <сообщение>` | Добавляет запись в лог дня |
| `Обнови статус в CLAUDE.md` | Обновляет секцию "Текущий статус" |
| `Покажи roadmap` | Показывает прогресс по роадмапу |
| `Закоммить и запуши` | Коммитит изменения и пушит на GitHub |

### Документация
| Команда | Что делает |
|---------|-----------|
| `Обнови архитектуру` | Обновляет docs/architecture.md |
| `Обнови roadmap` | Отмечает выполненные пункты |
| `Создай лог за сегодня` | Создаёт новый файл лога |

## Workflow: типичная сессия работы

```
1. Открыть проект:     cd C:\project21\konsensus && claude
2. Проверить статус:    "Продолжи работу над проектом"
3. Работать:            Давать задачи по одной
4. Логировать:          /log Сделал то-то
5. Коммитить:           "Закоммить изменения"
6. Пушить:              "Запуши на GitHub"
```

## Советы

### Как формулировать задачи
- **Хорошо:** "Создай страницу для списка споров в dashboard. Используй серверный компонент, данные из Supabase."
- **Плохо:** "Сделай фронтенд"

Чем конкретнее задача — тем лучше результат.

### Если что-то сломалось
```
Проект не собирается. Вот ошибка: [вставить ошибку]. Исправь.
```

### Если нужно вернуться назад
```
Откати последний коммит, он сломал [описание проблемы].
```

### Если нужно разобраться в коде
```
Объясни как работает [файл/компонент/функция].
```

## Telegram Bot — первичная настройка

### 1. Создать бота
Написать `@BotFather` в Telegram → `/newbot` → сохранить токен.

### 2. Добавить env-переменные в Vercel
```
TELEGRAM_BOT_TOKEN=<токен от BotFather>
TELEGRAM_WEBHOOK_SECRET=<любая случайная строка>
```

### 3. После деплоя — зарегистрировать webhook (один раз)
```bash
curl -X POST "https://api.telegram.org/bot<TOKEN>/setWebhook" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://konsensus.app/api/telegram","secret_token":"<WEBHOOK_SECRET>"}'
```

После этого бот готов к работе. Пользователи привязывают аккаунт через Профиль → «Получить код».

## Важно помнить

- **`.env.local`** — создай вручную из `.env.example` и заполни реальными ключами
- **Supabase** — нужно создать проект на supabase.com и вставить ключи в `.env.local`
- **Claude API** — нужен API key с console.anthropic.com
- **Логи** — ИИ будет логировать milestones автоматически в `docs/log/`
- **Коммиты** — ИИ коммитит только когда ты попросишь
- **Push** — ИИ напомнит пушить каждые 3-5 коммитов
