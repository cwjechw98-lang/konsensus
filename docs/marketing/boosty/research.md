# Boosty Research Notes

## Подтверждённые внешние наблюдения

### 1. Витрина Boosty

Официальный пост Boosty про витрину подтверждает:
- витрина берёт изображение из блока `Тизер поста`;
- если тизер не заполнен, карточка поста выглядит пустой;
- Boosty рекомендует квадратные изображения;
- платформа автоматически масштабирует обложку под `575 x 625 px`, поэтому важные элементы лучше держать в центре.

Источник:
- [Почему обложки на витрине — это важно](https://boosty.to/boosty/posts/d6ffc762-cf28-4a51-857f-6c6a6d38e564)

Практический вывод:
- первый витринный пост должен обязательно иметь отдельную teaser-cover;
- текст на обложке держать в центральной зоне;
- не делать длинные горизонтальные изображения для витрины.

### 2. Скрытые уровни и лимиты подписчиков

Официальный пост Boosty подтверждает:
- уровень можно скрыть;
- для уровня можно задать лимит подписчиков;
- лимит можно менять позже;
- скрытый уровень остаётся у текущих подписчиков, но недоступен новым;
- Boosty отдельно рекомендует использовать Telegram/Discord-чаты как ограниченный benefit для лимитированных уровней.

Источник:
- [Гибкое управление подписками на Boosty: скрытые уровни и лимиты подписчиков](https://boosty.to/boosty/posts/0f0bb27e-1446-4fbb-a82f-ee716460b9a9)

Практический вывод:
- для Konsensus это полезно позже, когда появится закрытая Telegram-группа;
- на старте лучше не обещать чат как benefit, пока не понятна модерация и реальный ритм общения.

### 3. Практические размеры графики

Официальной единой страницы с размерами всех блоков Boosty я не нашёл. Есть рабочий creator-guide с конкретными размерами:
- header: `1920 x 240 px`
- avatar: `450 x 450 px`
- teaser paid-post: `575 x 625 px`
- изображение уровня подписки: `240 x 150 px`
- изображения внутри поста: ширина `1080 px`

Источник:
- [Размеры графики и видео на Boosty](https://boosty.to/clubprofi/posts/8fb7d847-8c39-441b-9ef6-cfcb65df1ed2)

Это не официальный help-doc Boosty, поэтому использовать как практическую рекомендацию, а не как железный стандарт платформы.

## Telegram collectible username / Fragment

### Что подтверждено

Ручная проверка Fragment от `2026-03-16` показала:
- `@konsensus` найден на Fragment;
- статус: `Available`;
- minimum bid: `123 TON`.

Проверка выполнялась на:
- [Fragment](https://fragment.com/?query=konsensus)

Практический вывод:
- collectible username полезен для Telegram-идентичности проекта;
- он не заменяет собственный веб-домен для сайта.

### Важный нюанс

На Fragment также показывается отдельная логика для bot usernames: collectible username можно апгрейдить для назначения боту за дополнительную оплату. Это не то же самое, что просто владение collectible username.

## Что делать с этим сейчас

### Для Boosty

- стартовать с минимально жизнеспособного, но не пустого профиля;
- не перегружать страницу обещаниями;
- делать упор на:
  - поддержку инфраструктуры;
  - апгрейд ИИ;
  - прозрачный devlog проекта.

### Для Telegram username

- пока не выносить collectible username в основную первую цель Boosty;
- сначала решить, нужен ли именно Telegram collectible username или обычный веб-домен;
- если цель всё же появится, явно писать, что это Telegram-идентичность проекта, а не адрес сайта.

## Внутренний вывод для проекта

- Telegram release-posting уже есть и его можно использовать сразу;
- Boosty сейчас лучше заполнять как support + public devlog surface;
- отдельную автоматизацию постинга в Boosty пока не планировать, пока не подтверждён удобный API/workflow.
