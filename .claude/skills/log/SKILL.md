# Skill: /log

Добавляет запись в лог разработки проекта Konsensus.

## Использование

```
/log <сообщение>
```

## Поведение

1. Определить текущую дату (формат `YYYY-MM-DD`)
2. Проверить, существует ли файл `docs/log/YYYY-MM-DD.md`
   - Если нет — создать с шаблоном
   - Если да — добавить запись в существующий файл
3. Добавить сообщение в секцию "Что сделано" как `- [x] <сообщение>`

## Шаблон нового лога

```markdown
# Лог: YYYY-MM-DD

## Что сделано
- [x] <сообщение>

## Решения
- ...

## Следующие шаги
- [ ] ...

## Заметки
- ...
```

## Если файл уже существует

Добавить новый пункт `- [x] <сообщение>` в конец секции "Что сделано".

## Примеры

```
/log Настроена аутентификация через Supabase Auth
/log Создан компонент DisputeCard
/log Исправлен баг с отображением аргументов
```
